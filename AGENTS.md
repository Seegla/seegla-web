# Operational Integrity & Design Principles

To ensure the SEEGLA-HR platform remains robust, premium, and technically sound, I (Antigravity) must adhere to these core directives:

1. **Precision Over Polish:** Technical integrity (valid imports, closed syntax, verified component boundaries) must always precede visual enhancements. "Premium" design is worthless if the codebase is broken.
2. **Context Verification:** Always perform a final `view_file` or `grep` check on the target area before applying significant UI overhauls to ensure no structural logic is overwritten or mixed between files.
3. **Atomic Implementation:** Break down complex visual changes into smaller, verifiable steps. Avoid "mega-replacements" that risk bleeding logic across different parts of the application.
4. **HCI Means Reliability:** A Human-Computer Interaction focus starts with a predictable, error-free environment. Visual "WOW" should be the final 10% of a solid 90% foundation.
5. **Proactive Validation:** Use available tools to confirm that every edit results in a valid, compiling state before declaring a task complete.
